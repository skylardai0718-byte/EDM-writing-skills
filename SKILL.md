# EDM 营销撰写 Skill

> **适用对象**：硬件产品、科技消费品、NAS / homelab / 本地 AI 等需要"教育用户"才能转化的品类。  
> **核心理念**：文案不靠折扣驱动，而是靠场景、教程、社区感和品牌信任共同运转。

---

## 前置：知识库构建（Context Building）

**在写任何 EDM 之前，先构建品牌知识库。** 这是保证文案语气统一、"甲方想看"的核心步骤。

### Step 0 — 用 NotebookLM 提取产品知识

把以下内容喂给 [Google NotebookLM](https://notebooklm.google.com/):
- 表现好的 KOL YouTube 视频（科技/生活/垂类）
- 官网产品页
- 品牌 Blog / 开发日志 / 更新日志
- 已发布的 PR 稿 / 媒体报道

然后用 ChatGPT 生成如下 Prompt，发到 NotebookLM，要求它输出以下 8 个知识库文件：

```
请基于所有资料，生成以下8个文件：
1. [品牌名] Trust & Social Proof Library    — 社区规模、用户评价、媒体背书、可引用数据
2. [品牌名] EDM Objection Handling Library  — 常见购买顾虑 + 反驳话术
3. [品牌名] Content Asset Index             — 可用内容资产（教程、视频、案例）清单
4. [产品名] EDM Master Brief               — 产品定位、核心卖点、目标受众、差异化
5. [品牌名] EDM Voice Guide                — 品牌语气、偏好词汇、句式模板、禁忌写法
6. [品牌名] EDM Use Case Library           — 用户使用场景库（按人群细分）
7. [品牌名] EDM Best Practices & Postmortems — 历史高/低表现邮件分析
8. Competitor EDM Benchmark for [品牌名]   — 竞品邮件策略对比
```

> **为什么这样做**：这些文件全部基于已被团队审核过的内容（官网/PR/产品页），确保 AI 之后写出的用词和语气对齐"甲方认可的标准"，而不是凭空生成。

### Step 0.5 — 在 VS Code 中配置 Claude Code + Skills

在 VS Code 中运行 Claude Code（需安装 GitHub Copilot Chat 扩展），安装以下三个 Skills：
- `content-strategy` — 内容规划
- `edm-marketing`（本文件）— EDM 撰写执行
- `marketing-ideas` — 创意灵感触发

将上一步生成的 8 个知识库文件放入工作区根目录，作为 AI 写作时的上下文参考。

## 核心工作流（四步撰写）

### Step 1 — 🧠 场景拆分 & EDM 策略

**触发词**: "帮我策划这次的EDM"、"我要写发售邮件"、"规划一个邮件Flow"

**AI 应执行**:

1. 确认以下信息（向用户提问，或从知识库文件中提取）：

   | 信息项 | 说明 |
   |--------|------|
   | 产品/活动名称 | 核心卖点是什么 |
   | 目标受众 | 购买阶段：新订阅 / 兴趣用户 / 已购买 |
   | 邮件类型 | 单封推送 / Flow 序列 / 节日促销 |
   | 时效性 | 是否有限时优惠、库存、截止时间 |

2. 根据目标选择邮件结构：

   | 结构 | 适用场景 | 核心逻辑 |
   |------|---------|---------|
   | **A: 事件先行** | 发售、上新、价格更新 | 先说"发生了什么" → 卖点解释 → 行动信息 |
   | **B: 场景先行** | 教程、项目型、homelab | 场景 → 痛点 → 解决方案 → offer |
   | **C: 社区内容型** | 双周报 / Newsletter | 内容 roundup → 多模块 → 顺带转化 |
   | **D: 会员庆典型** | 节日、会员日、促销 | 感谢/身份感 → 活动权益 → 产品 → 社区 tone |

3. 输出策略文档：

```markdown
# EDM 策略 — [品牌/产品名] [主题]

## 邮件定位
- 类型: [单封推送 / Flow第X封 / 节日促销]
- 结构选择: [A/B/C/D] — 理由: [...]
- 受众: [购买阶段 + 特征标签]

## 核心信息
- 主要卖点:
- 时效信息（如有）:
- 信任背书（引自知识库）:

## 推荐邮件角度（2-3个）
1. [角度名] — [一句话描述] — 预期效果
2. [角度名] — [一句话描述] — 预期效果
3. [角度名] — [一句话描述] — 预期效果
```

---

### Step 2 — ✍️ 生成 EDM 初稿

**触发词**: "好，用角度X写初稿"、"开始写"

**AI 应按以下模块生成**（参考品牌 Voice Guide 的语气规范）:

```markdown
# EDM 初稿 — [主题]

## 主题行（3个变体，注明策略）
1. [标题] — 策略：[时间感/利益点/身份感/动作感]，推荐理由：[...]
2. [标题] — 策略：[...]
3. [标题] — 策略：[...]

## 预览文案（Preview Text）
→ 不重复标题，补充一条具体信息（奖励/场景/限时理由）
[预览文案]

## 邮件正文

### Hero / 开场（1-2句）
[内容] — 用场景痛点或事件引入，不用产品名堆砌

### 主体段落
[按选定结构 A/B/C/D 展开]

### 核心卖点模块（可选，3个以内）
[每个卖点配具体使用场景，而非空泛描述]

### CTA 按钮文案（3个变体）
1. [按钮] — 风格：探索型
2. [按钮] — 风格：行动型
3. [按钮] — 风格：身份认同型

## 页脚
[标准合规页脚：退订链接 + 公司地址 + 隐私政策]
```

**通用写作原则**（适用于硬件/科技品类）:
- 句式短促有力，结构为"场景痛点 + 硬件解法 + 用户掌控感"
- CTA 要和正文模块一一对应，用户知道点击后会去哪里
- 避免：空泛形容词（Revolutionize / Cutting-edge / Perfect）、说教式写法（"You should..."）
- 优先：具体数字、真实社区数据、用户案例

### Step 3 — 🚀 优化（提升打开率/点击率/降低退订）

**触发词**: "优化这份文案"、"打开率太低怎么改"、"退订率太高"

**AI 应执行转化优化检查**:

```markdown
## 转化优化分析

### 标题检查
- [ ] 是否包含"时间感/利益点/身份感/动作感"至少之一？
- [ ] Preview Text 是否补充了标题未说的具体信息？
- [ ] 两者组合是否回答了"我为什么现在要点开"？

### 正文检查
- [ ] 开场是否用场景/事件引入（而非产品名堆砌）？
- [ ] 是否有具体数字/社区规模/用户评价作为信任支撑？
- [ ] 是否包含使用场景/教程/项目灵感（而非纯促销信息）？
- [ ] CTA 是否具体且与对应模块内容一致？

### 退订风险检查
- [ ] 这封是否在连续促销序列中？前后有无内容/关系型邮件中和？
- [ ] 邮件频率是否合理？（Cart Abandon Flow 建议上限 3-4 封）
- [ ] 是否排除了已购买该产品的用户？（避免重复推送）

### A/B 测试建议
1. Subject Line: [变体A] vs [变体B]
2. CTA 文案: [变体A] vs [变体B]
3. 正文结构: [事件先行] vs [场景先行]

### 性能目标参考（硬件/科技品类）
- 打开率目标: >25%
- 点击率目标: >2.5%
- 退订率警戒线: <0.5%
```

---

### Step 4 — ✅ 审校 & 发送清单

**触发词**: "最后审校"、"准备发送"、"帮我检查"

```markdown
## 发送前最终检查

### 语言 & 品牌合规
- [ ] 无拼写/语法错误（中英文均检查）
- [ ] 语气符合品牌 Voice Guide（拒绝空泛、说教、超级企业风）
- [ ] 信任数据引自已审核的 Trust & Social Proof Library
- [ ] 未使用未经验证的夸大词（"全球首款"等）

### 技术检查
- [ ] 所有 CTA 链接可用且带 UTM 参数
- [ ] 图片有 ALT 文本（无图片环境下仍可阅读）
- [ ] 移动端预览正常
- [ ] 退订链接 & 隐私政策链接存在

### 发送决策
- [ ] 已在内部发送测试邮件
- [ ] 已排除"已购买该产品"的用户（Flow 隔离）
- [ ] 确认发送时间（避开周一早 9 点/周五下午等高噪声时段）
- [ ] 如为 A/B 测试，确认测试组比例和胜者判定标准
```

## Flow 序列设计参考

当需要设计多封邮件序列时，使用以下框架：

### Welcome Flow（新订阅用户）
```
Email 1（即时）: 品牌价值观 + 社区欢迎 — 无销售压力
Email 2（D3）:   核心使用场景介绍（教程入口）
Email 3（D7）:   社交证明 + 社区故事
Email 4（D14）:  首次产品展示（附使用场景，非纯参数）
```

### Post-Purchase Flow（已购买用户）
```
Email 1（即时）: 订单确认 + 开箱/上手指引
Email 2（D2）:   快速上手教程（引 Content Asset Index 中的资源）
Email 3（D7）:   进阶功能 / 社区挑战
Email 4（D30）:  配件推荐 / 用户故事征集
```

### 培育 Flow（兴趣用户/未购买）
```
阶段一（兴趣）:  教程/场景内容为主，不急于推销
阶段二（考虑）:  社区规模 + 信任背书 + 与竞品对比
阶段三（决策）:  限时优惠 + 异议处理（引 Objection Handling Library）
```

## 高表现邮件公式（基于真实数据提炼）

### 标题公式
> `[明确事件/状态] + [时效/利益/身份感]`

✅ 好标题：
- `[产品] Pre-Order Is Live – Limited $50 Coupon Inside`
- `A Weekend Project for Your Home Server`
- `Member Day Starts Now: Discounts + Giveaways`

❌ 差标题特征：只有品牌名 + 空泛口号，无时效/无具体利益。

### Preview Text 公式
> 标题讲主题，Preview 补一层具体信息（奖励/场景/限时理由），不重复标题。

### CTA 公式
> 与模块内容一一对应，非通用"Learn More"。  
> 例：`Pre-order Now` / `Immich Setup Tutorial` / `Apply for Pioneer Program`

## 配套 Skills（推荐同时安装）

本 Skill 专注 EDM 撰写执行，完整工作流还需要以下两个 Skills 配合：

| Skill | 用途 | 何时调用 |
|-------|------|---------|
| `content-strategy` | 内容日历规划、主题优先级排序、内容支柱设计 | 在写单封 EDM 之前，规划整体内容节奏 |
| `marketing-ideas` | 提供 139 个经过验证的营销创意，按预算/阶段/类别组织 | 卡创意、找不到差异化角度时 |

### 安装方式

在你的项目 `.github/skills/` 目录下分别创建对应文件夹和 `SKILL.md`，或直接使用支持 Skills 的 VS Code 扩展（GitHub Copilot Chat）自动发现。

### 三个 Skill 的分工

```
content-strategy   →  规划"写什么"（主题/时间/受众优先级）
        ↓
edm-marketing      →  执行"怎么写"（本文件，四步撰写流程）
        ↑
marketing-ideas    →  补充"有哪些角度"（卡壳时随时调用）
```
